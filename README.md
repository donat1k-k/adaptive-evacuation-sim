# adaptive-evacuation-sim

Программное моделирование и сравнительный анализ алгоритмов адаптивной маршрутизации при эвакуации людей из здания в условиях динамически изменяющейся среды.

## Что это

**Исследовательский стенд**, а не игра, не проект по ОБЖ и не обычный сайт. Программа моделирует эвакуацию, сравнивает алгоритмы маршрутизации по количественным метрикам, собирает данные и строит выводы. Ценность — в данных (метрики, серии, кривая эвакуации), а не в картинке.

## Научный фокус

Информатика: 2D-сетка как граф, алгоритмы поиска пути (BFS/A*), агентное моделирование, динамические веса клеток, вычислительный эксперимент, метрики, статистика, воспроизводимость.

Главный исследовательский вопрос: **при каких условиях адаптивная маршрутизация эффективнее простых стратегий эвакуации?** Сильный результат — найти и количественно показать точку, где адаптив перестаёт выигрывать.

## Текущий статус

**Готовы E0–E7.** Модели данных (E2) и headless-движок симуляции (E3, детерминирован по seed) реализованы. На E4 добавлена **базовая отладочная визуализация** движка. На **E5** реализованы три настоящих алгоритма маршрутизации за единым интерфейсом `MovementPolicy` (общий A*): **A1** Nearest Exit, **A2** Shortest Path A*, **A4** Adaptive Weighted A* (`cost = base + α·density + β·danger + γ·smoke`). На **E6** динамические события среды доведены до MVP: `block-cell`, `block-exit`, `hazard-appear` применяются детерминированно до движения тика и закреплены selfcheck'ами; A1 на блокировке своего выхода упрямо застревает, A2/A4 перенацеливаются на альтернативный выход. На **E7** добавлен расчёт **метрик** (`src/metrics`, чистый headless-слой из финального состояния): % эвакуации, makespan, среднее/мин/макс время (только по эвакуированным), reroutes, нагрузка на выходы, кривая эвакуации. В demo (`npm run dev`) — **селектор алгоритма A1/A2/A4** и базовые метрики прогона; после события `block-exit` (тик 6) видно разницу реакции.

> **Важно:** demo-карта `demoE4` — отладочная, **не** финальный эксперимент (сценарии S1–S3 — этап E8, метрики — E7). Делать исследовательские выводы «какой алгоритм лучше» по этому демо нельзя: метрики ещё не считаются. Коэффициенты A4 — черновые (подбор — E12).

## Как развивается проект

По этапам из `PROJECT_PLAN.md`: модель данных → headless-движок симуляции → базовая визуализация → алгоритмы A1/A2/A4 → динамические события → метрики → сценарии → графики/экспорт → MVP → серии экспериментов и анализ → Strong Version → итоговая работа.

Принцип: сначала корректное ядро и данные, потом оформление. Карты — как готовые сценарии/JSON, без редактора в MVP.

## Стек

Vite + React 19 + TypeScript (strict). Дальше — Canvas-визуализация, экспорт CSV/JSON. Без backend, без базы данных. Подробности — в `docs/PROJECT_SPEC.md §23`.

## Структура `src/`

`app/` — корневой компонент · `components/` — UI · `models/` — типы данных · `simulation/` — headless-движок · `algorithms/` — A1/A2/A4 + общий A* (E5) · `scenarios/` — карты-данные · `metrics/` — метрики · `export/` — CSV/JSON · `utils/` — вспомогательное. Папки `metrics/export` пока заглушки (см. `docs/PROJECT_PLAN.md §10`).
