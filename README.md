# adaptive-evacuation-sim

Программное моделирование и сравнительный анализ алгоритмов адаптивной маршрутизации при эвакуации людей из здания в условиях динамически изменяющейся среды.

## Что это

**Исследовательский стенд**, а не игра, не проект по ОБЖ и не обычный сайт. Программа моделирует эвакуацию, сравнивает алгоритмы маршрутизации по количественным метрикам, собирает данные и строит выводы. Ценность — в данных (метрики, серии, кривая эвакуации), а не в картинке.

## Научный фокус

Информатика: 2D-сетка как граф, алгоритмы поиска пути (BFS/A*), агентное моделирование, динамические веса клеток, вычислительный эксперимент, метрики, статистика, воспроизводимость.

Главный исследовательский вопрос: **при каких условиях адаптивная маршрутизация эффективнее простых стратегий эвакуации?** Сильный результат — найти и количественно показать точку, где адаптив перестаёт выигрывать.

## Текущий статус

**Готовы E0–E10.** Модели данных (E2), headless-движок (E3, детерминирован по seed), базовая визуализация (E4), три алгоритма за интерфейсом `MovementPolicy` (E5): **A1** Nearest Exit, **A2** Shortest Path A*, **A4** Adaptive Weighted A* (`cost = base + α·density + β·danger + γ·smoke`). Динамические события (E6): `block-cell`/`block-exit`/`hazard-appear`. Метрики (E7, `src/metrics`): % эвакуации, makespan, время (только по эвакуированным), reroutes, нагрузка на выходы, кривая эвакуации.

На **E8** добавлены три финальных сценария-как-данные (`src/scenarios`, реестр `FINAL_SCENARIOS`): **S1** baseline (открытая комната, 2 выхода, без событий), **S2** bottleneck (разделяющая стена с двумя проходами, 24 агента → пробка), **S3** dynamic (задымление + блокировка выхода во время прогона). На **E9** — headless **comparison runner** (`src/comparison`): прогон `scenario × algorithm × seed`, paired-by-seed, агрегация (mean evacuatedFraction/makespan/времени/reroutes/stranded). На **E10** — **экспорт** CSV/JSON (`src/export`) с полным конфигом+seed; в demo (`npm run dev`) — debug-секция сравнения/экспорта со скачиванием результатов.

Запуск всех sanity-проверок: **`npm run selfcheck`** (Node исполняет TypeScript напрямую, без новых зависимостей) — **57 PASS** (sim/algo/metrics/scenarios/comparison/export).

> **Важно:** demo-карта `demoE4` — отладочная, **не** финальный эксперимент (исследовательские сценарии — S1/S2/S3). Графиков (E11) и исследовательских выводов пока нет: E8–E10 готовят **данные**, а не заключения. Коэффициенты A4 — черновые (подбор — E12).

## Команды

```bash
npm install      # установка зависимостей
npm run dev      # dev-сервер (http://localhost:5173)
npm run build    # строгая проверка типов (tsc -b) + production-сборка в dist/
npm run preview  # просмотр production-сборки
npm run lint     # ESLint
npm run typecheck # только проверка типов
npm run selfcheck # все sanity-проверки (Node исполняет TS напрямую, без доп. зависимостей)
```

## Как развивается проект

По этапам из `PROJECT_PLAN.md`: модель данных → headless-движок симуляции → базовая визуализация → алгоритмы A1/A2/A4 → динамические события → метрики → сценарии → графики/экспорт → MVP → серии экспериментов и анализ → Strong Version → итоговая работа.

Принцип: сначала корректное ядро и данные, потом оформление. Карты — как готовые сценарии/JSON, без редактора в MVP.

## Стек

Vite + React 19 + TypeScript (strict). Дальше — Canvas-визуализация, экспорт CSV/JSON. Без backend, без базы данных. Подробности — в `docs/PROJECT_SPEC.md §23`.

## Структура `src/`

`app/` — корневой компонент · `components/` — UI · `models/` — типы данных · `simulation/` — headless-движок · `algorithms/` — A1/A2/A4 + общий A* (E5) · `scenarios/` — сценарии S1/S2/S3 + реестр/конфиг/валидатор (E8) · `metrics/` — метрики (E7) · `comparison/` — comparison runner (E9) · `export/` — CSV/JSON (E10) · `utils/` — геометрия + валидатор сценария. Раннер проверок — `scripts/selfcheck.ts`. Папка `experiment/` — пока заглушка (E12, см. `docs/PROJECT_PLAN.md §10`).
