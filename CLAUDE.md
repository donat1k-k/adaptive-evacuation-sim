## Claude Code specific

- Роли: **архитектор** (структура, интерфейсы, дизайн движка/экспериментов, сложные решения), **исполнитель** на раннем этапе (пока один инструмент), **ревьюер** и **дебаггер** (детерминизм, контеншен, корректность метрик).
- Перед крупными изменениями работать **как в Plan Mode**: сначала план, потом код.
- **Не начинать реализацию без подтверждения пользователя после плана.**
- Не тратить контекст на переписывание документации без необходимости.
- После этапов обновлять `docs/HANDOFF.md`; сложные решения — в `docs/DECISIONS.md`.
- Новая сессия начинается с чтения `docs/PROJECT_SPEC.md` + `docs/HANDOFF.md` (при необходимости `docs/PROJECT_PLAN.md`, `docs/DECISIONS.md`).
